01 — Components

Interactive elements

Buttons, cards, modals, and toasts — each animated with purpose. Hover states gate-checked for mobile. Press feedback on every clickable element.

↑
Ease-Out Entrances
Start fast, decelerate into place. Feels immediate and responsive — never sluggish.
150–250ms
↓
Ease-In Exits
Accelerate away from the frame. Faster than entrances. Exit 200ms vs Enter 300ms.
faster
◉
Transform Only

Scale on Press
Buttons scale to 0.97 on :active. Makes the UI feel physical and responsive.
physicality
◇
No Scale from Zero
Nothing appears from nothing. Start from scale(0.95) + opacity 0 for modals.
realism

Staggered Reveals
30–60ms between items. Never more than 100ms stagger. Limit to 3–7 elements.

02 — Timing

The clock doesn't lie

Duration decisions follow frequency of use. Hover interactions (tens/day): barely animate. Modals (occasional): standard duration. Never exceed 300ms for UI interactions.

Micro-interaction100ms
Hover feedback150ms
Standard220ms
Complex transition350ms

03 — Principles

Why it matters

The best animations are the ones you don't notice. Motion should make software feel instantaneous and physical — not decorative. Taste is trained.

∞
Invisible Motion
Users notice the interface feels good — not that it's animated. That's the goal.
⚖
Purposeful Speed
Animation should never make a user wait. If it slows interaction, cut the duration.
🌍
Physics-Based
Motion mimics real-world physics — acceleration, deceleration, gravity.
♿
Reduced Motion
prefers-reduced-motion respected. Movement stops, opacity transitions remain.
